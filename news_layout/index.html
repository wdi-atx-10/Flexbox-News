Here's a cool toy I've been working on. It's in a thing called "processing" which is perhaps the worst possible name for a programming language I've ever heard. On the other hand, it's a fun java like environment that makes it easy to construct graphical animations right outside of the box. Here I've made some balls that bounce and also decelerate and fall down to the bottom with gravity. The line is a vector caused by the mouse clicking and dragging, shooting new balls out relative to that vector. I worked a bit on the collision detection which I got "sortof" working. I put the project on hold when I found out that I had to read a few page math paper to get the balls to bounce off each other in relation to their proper vectors (like billiards).

This is called trippy dots! I like this little visual for a couple of reasons. First, if you make little things like this you might be able to put them in the background of webpages, or use them as static icons and backgrounds for visual effects that css may not be able to do on its own (or as easily). This is great if you want to differentiate your website from what everyone else is doing these days. Also, it's interesting to note that the programming for this is very simple, but tweaking the dots to get the stereoscopic overlapping effect is a matter of artistic finesse. So it's a great toy to play with and get to exercise artistic ability over technical.

Why do I like these languages? First, python. Python is easy to learn and there's a book out there by no starch press called Python Crash Course. It's only 150 or so pages and you can pretty much knock out the basics of the language quickly. It's also a high level language, meaning that it is highly abstracted away from the compiler. It lets you do more things, at the expense of doing them slower and sometimes not being as precise. But! You can write very readable and clean code. Both Rust and Go are newer languages that are compiled rather than interpreted. Go is more useful for concurency, which is a way to tell a computer to break a problem into several steps and compute each step separately. This is greate for cloud computing and multi-core machines, so long as the problem at hand is easily parallelized. Personally I prefer Rust. While Rust can also handle concurency what's so neat about it is that it's error handler is very verbose (which is great for n00bs) and it's mutability abstraction prevents pointer loss errors which can be a problem in C.

I've only done a few projects in each of the languages, mostly just to test the waters. I have a cool 30 or 40 line Conway's game of life in native C, but I don't really think I can count that one. The one's I'm most proud of at the moment have been making a sudoku solver in Go, making a Connect4 game and tower of Hanoi solver in Rust, and probably the tictactoe game I made in python. Both the tictactoe and the Connect4 game have "ai" which is fancy speak for a decision tree for automated moves on the computer's part. The two solvers, for sudoku and towers of Hanoi, use an algorithm to generate the result. While the towers of Hanoi code is super clean, that I algo only implemented from wikipedia. The sudoku one I got on my own and it's actually pretty shwifty. Future projects need to start implementing more visual components, as all these games are only playable through terminal.
